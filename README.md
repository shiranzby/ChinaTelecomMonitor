# TelecomMonitor

中国电信手机话费、流量、语音通话监控工具。纯 Python 实现，无需 Docker，一个脚本搞定全量查询 + 多渠道通知推送。

## 相比同类方案的改进

相比常见的电信监控脚本，本工具在以下方面做了增强：

| 方面 | 常见方案 | 本工具 |
|------|---------|--------|
| **查询数据** | 仅余额、流量等基础项 | 余额、欠费、消费明细、通话（含分类）、通用/专用/总流量、流量包明细、共享套餐用量 |
| **登录安全** | 无保护 | 连续失败 5 次自动停止，防止账号被风控锁定 |
| **通知推送** | 仅 1-2 种渠道 | 9 种主流渠道，配置驱动，填了就推送、不填就跳过 |
| **输出控制** | 固定字段 | 全局默认 + 每号独立覆盖，按需开关任意字段 |
| **数据完整性** | API 直接展示 | 当 API 未返回通用/专用流量总量时，自动从流量包明细反算补全 |
| **部署方式** | 依赖 Docker / 青龙面板 | 零依赖部署，双击 BAT 或定时任务即可运行，也支持 Docker |

## 功能特性

- **全量数据查询** — 余额、欠费、本月消费、费用明细、通话用量（含分类）、通用/专用/总流量、流量包明细、共享套餐用量
- **登录风控保护** — 连续失败 5 次自动停止，防止账号被风控锁定
- **多渠道通知推送** — 支持 9 种主流推送渠道，配置文件驱动，填了就推送、不填就跳过
- **全中文配置** — JSON 配置文件全部使用中文键名，直观易读
- **灵活输出控制** — 全局默认 + 每号独立覆盖，按需开关任意字段
- **Token 持久化** — 登录凭证自动缓存，避免频繁登录触发风控
- **多号批量查询** — 配置文件一次配好，双击 BAT 即可查全部

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置账号

```bash
cp telecom_config.example.json telecom_config.json
```

编辑 `telecom_config.json`，填入手机号和服务密码：

```json
{
  "手机号": [
    { "号码": "18000000000", "服务密码": "your_password" }
  ]
}
```

> 服务密码为电信 6 位数字服务密码，非登录密码。如忘记可在电信 APP 或拨打 10000 重置。

### 3. 运行查询

```bash
# 从配置文件读取所有号码
python telecom_query.py

# 单号查询
python telecom_query.py --phone 18000000000 --password your_password

# JSON 格式完整数据输出
python telecom_query.py --json

# 强制显示流量包明细
python telecom_query.py --packages
```

### Windows 用户

双击 `查询话费.bat` 即可查询配置文件中的所有号码。

## 配置说明

配置文件 `telecom_config.json` 支持以下字段：

### 账号配置

| 字段 | 说明 |
|------|------|
| `号码` | 手机号 |
| `服务密码` | 电信 6 位数字服务密码 |
| `输出设置` | 可选，该号码的独立输出开关，优先级高于全局 |

### 输出设置

| 字段 | 默认 | 说明 |
|------|------|------|
| `城市` | 1 | 所在省市 |
| `余额` | 1 | 账户余额 |
| `欠费` | 0 | 欠费金额 |
| `本月消费` | 1 | 当月已产生费用 |
| `费用明细` | 0 | 套餐/增值业务等逐项构成 |
| `通话已用` | 0 | 已用/总量（分钟） |
| `通话剩余` | 0 | 剩余分钟数 |
| `通话分类` | 0 | 省内/国内等分类明细 |
| `通用流量` | 1 | 通用流量已用/总量 |
| `专用流量` | 1 | 专用流量已用/总量 |
| `总流量` | 1 | 总流量已用/总量 |
| `流量包明细` | 0 | 每行一条流量包 |
| `账单提示` | 0 | 账单周期等提示 |
| `查询时间` | 1 | 查询时间戳 |

> 值为 `1` 显示，`0` 隐藏。

## 通知推送

在配置文件中添加 `通知推送` 段，设置 `"启用": true` 后，脚本会在查询完成后自动推送结果。

### 基础配置

```json
{
  "通知推送": {
    "启用": true
  }
}
```

- `启用`：是否开启通知推送（默认 `false`）

### 支持的推送渠道

填入对应字段即可自动启用，不填则跳过。支持同时启用多个渠道。

| 渠道 | 配置字段 | 说明 |
|------|---------|------|
| **SMTP 邮件** | `SMTP服务器` `SMTP端口` `SMTP_SSL` `发件邮箱` `邮箱密码或授权码` `收件邮箱` | 最通用的推送方式 |
| **PushPlus** | `PushPlus令牌` | 微信推送，填令牌即可 |
| **Server酱** | `Server酱密钥` | 微信推送，支持 Turbo 版 |
| **Bark** | `Bark设备码` | iOS 推送 |
| **Telegram** | `Telegram机器人Token` `Telegram用户ID` | Telegram Bot 推送 |
| **企业微信** | `企业微信机器人Key` | 机器人 Webhook 推送 |
| **钉钉** | `钉钉机器人Token` `钉钉机器人密钥` | 支持签名验证 |
| **飞书** | `飞书机器人Key` | 机器人 Webhook 推送 |
| **自定义 Webhook** | `自定义Webhook地址` `自定义Webhook方法` `自定义Webhook请求体` | 通用兜底，支持模板变量 `$title` `$content` |

### 推送配置示例

```json
{
  "通知推送": {
    "启用": true,
    "推送渠道": {
      "SMTP服务器": "smtp.qq.com",
      "SMTP端口": 465,
      "SMTP_SSL": true,
      "发件邮箱": "your@qq.com",
      "邮箱密码或授权码": "your_auth_code",
      "收件邮箱": "receiver@example.com"
    },
    "可选渠道": {
      "PushPlus令牌": "your_pushplus_token",
      "企业微信机器人Key": "your_wecom_key"
    }
  }
}
```

## Docker 部署

项目根目录下提供 `Dockerfile`，可直接打包为镜像运行。配置文件和 Token 缓存通过挂载目录 `/app/data` 持久化。

**构建与运行：**

```bash
# 构建镜像
docker build -t telecom-monitor .

# 运行（挂载配置文件目录）
docker run --rm \
  -v /path/to/your/config:/app/data \
  telecom-monitor
```

> 将 `telecom_config.json` 放在宿主机挂载目录中，脚本会自动从 `/app/data/telecom_config.json` 读取。Token 缓存也会保存在该目录下。也可通过 `-e TELECOM_DATA_DIR=/other/path` 指定其他数据目录。

**定时任务（cron）：**

```bash
# 每天早上 8 点自动查询并推送
0 8 * * * docker run --rm -v /path/to/your/config:/app/data telecom-monitor
```
